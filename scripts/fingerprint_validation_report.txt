  All φ_attn and planner decisions match paper claims: ✅ YES

Root-Cause Analysis:

  YOLO11s:
    - Paper claims φ_attn = 0 (dense-conv only).
    - Real weight has φ_attn = 0.1477 because the C2PSA block (index 10)
      contains an Attention module with Conv2d-based qkv/proj/pe submodules.
      The fingerprint counts every submodule whose name contains 'attn',
      inflating the ratio from 0 to 0.15.

  YOLO12s:
    - Paper claims φ_attn ≈ 0.45 (moderate attention).
    - Real weight has φ_attn = 0.8667 because the A2C2f blocks contain
      many deeply-nested AAttn modules (qkv, proj, pe), each with Conv2d/BN/Act
      children. The string-based name matching counts every child, giving
      104 attn-named modules out of 120 total conv+linear modules.
      This is far higher than the paper's simplified mock model (4/9 ≈ 0.45).

  RT-DETR-l:
    - Paper claims φ_attn ≈ 0.85 (pure attention).
    - Real weight has φ_attn = 0.2121 because the ResNet backbone contributes
      122 Conv2d layers while the Transformer decoder only contributes 42
      attention-named modules (self_attn, cross_attn, and their Linear children).
      The paper's mock model (3 conv + 1 linear = 4 total, 3 attn) is not
      representative of the full architecture.

Conclusion:
  The ArchitectureFingerprint.compute() implementation is NOT representative
  of real model architectures. It uses string-based name matching on all
  named modules, including submodules of attention blocks, which massively
  inflates φ_attn for YOLO12s and under-reports it for RT-DETR-l relative
  to the paper's simplified mock-model claims. The paper's mock models
  (used in test_planner.py) do not reflect the real module hierarchy.
